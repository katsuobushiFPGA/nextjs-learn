# Next.js における画像の最適化：技術概要と実装ガイドライン

## エグゼクティブ・サマリー

Next.js の `<Image>` コンポーネントは、従来の HTML `<img>` 要素を高度に拡張したものであり、Web パフォーマンスの向上とユーザー体験の最適化を自動化するツールとして設計されている。

主な特徴：
- 最新の画像フォーマット（WebP 等）の採用によるファイルサイズの削減
- レイアウトシフトの防止による視覚的安定性の確保
- ブラウザのネイティブ遅延読み込みを活用したページロードの高速化

ローカルおよびリモートの双方のアセットに対して、最小限の設定で高度な最適化を適用することが可能。

---

## 1. `<Image>` コンポーネントの主要機能

| 機能 | 概要 |
|------|------|
| **サイズの最適化** | デバイスごとに適切なサイズの画像を自動配信。WebP 等のモダンフォーマットでデータ転送量を削減 |
| **ビジュアルの安定性** | 画像読み込み時のレイアウトシフト（Layout Shift）を自動防止 |
| **ページロードの高速化** | ネイティブ Lazy Loading でビューポート直前にのみ読み込み。ぼかしプレースホルダーも利用可能 |
| **アセットの柔軟性** | オンデマンドリサイズ対応。ローカル・リモート問わず最適化可能 |

---

## 2. ローカル画像の管理と実装

### 2.1 静的ファイルとインポート

- **保存場所**：ルートディレクトリの `public/` フォルダに保存する
- **参照方法**：ベース URL（`/`）から始まるパスで直接参照できる
- **自動サイズ検知**：静的インポートした場合、`width` と `height` の固有値を自動取得する

### 2.2 累積レイアウトシフト（CLS）の防止

静的インポートで取得した画像サイズ情報はアスペクト比の確定に使用される。これにより、画像読み込み完了前にブラウザが適切なスペースを確保できるため、CLS を回避できる。

---

## 3. リモート画像の管理とセキュリティ設定

### 3.1 必須プロパティ

Next.js はビルド中にリモートファイルにアクセスできないため、以下を手動で指定する必要がある。

- **寸法指定**：`width` / `height` を手動指定するか、`fill` プロパティで親要素に合わせる
- **プレースホルダー**：`blurDataURL` を指定するとロード中の視覚的フィードバックを強化できる

### 3.2 セキュリティ設定（`next.config.js`）

悪意のある使用を防ぐため、許可するリモート画像ソースを制限する。

| 設定項目 | 説明 | 例 |
|----------|------|----|
| リモートパターン | 許可する画像の URL パターンを指定 | 特定の AWS S3 バケットなど |
| 具体性の確保 | ワイルドカードを避け、可能な限り具体的なパスを指定 | `example-bucket.s3.amazonaws.com` |

---

## 4. 結論

Next.js の画像最適化機能は、複雑な画像処理ロジックを記述することなく高いパフォーマンス基準を満たすことを可能にする。

- **ローカル画像**：自動化の恩恵を最大限活用する
- **リモート画像**：適切な設定とセキュリティ構成を施す

詳細なプロパティや API の挙動については、公式の API リファレンスを参照すること。
