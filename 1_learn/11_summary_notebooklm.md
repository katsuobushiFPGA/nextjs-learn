# Next.js 16 における CSS スタイリング概要

## エグゼクティブ・サマリー

Next.js 16 は以下の CSS スタイリング手法をネイティブでサポートしている。

- Tailwind CSS
- CSS Modules
- Global CSS
- 外部スタイルシート
- Sass
- CSS-in-JS

**基本方針：** Tailwind CSS を優先的に使用し、ユーティリティクラスで対応しきれないコンポーネント固有のスタイルには CSS Modules を併用する。

---

## 主要なスタイリング手法

### 1. Tailwind CSS

ユーティリティファーストな設計で、低レベルのユーティリティクラスを組み合わせてデザインを構築する。

**セットアップの手順：**
1. `postcss.config.mjs` にプラグインを追加
2. グローバル CSS にインポート
3. ルートレイアウト（`app/layout.tsx`）に統合

> 非常に古いブラウザのサポートが必要な場合は Tailwind CSS v3 のセットアップ手順を参照。

### 2. CSS Modules

クラス名を自動的にユニークな文字列へ変換することで CSS をローカルスコープ化する。

- **利点：** 異なるファイル間でクラス名が重複しても競合が発生しない
- **使用法：** `.module.css` 拡張子のファイルを作成し、コンポーネント内でインポートして使用する

```css
/* blog.module.css */
.blog {
  padding: 24px;
}
```

```tsx
import styles from './blog.module.css'

export default function Page() {
  return <main className={styles.blog}></main>
}
```

### 3. グローバル CSS

`app/layout.tsx` にインポートすることで全ルートにスタイルを適用する。

**注意：** React の Suspense 統合の仕様により、ルート間移動時にスタイルシートが削除されず競合が生じる可能性がある。Tailwind のベーススタイルなど、真にグローバルなスタイルのみに限定することが推奨される。

### 4. 外部スタイルシート

`app` ディレクトリ内の任意のコンポーネントで外部パッケージの CSS をインポートできる。React 19 以降では `<link rel="stylesheet" href="..." />` の直接記述もサポートされている。

---

## CSS の順序制御と最適化

### 順序の決定要因

CSS の適用順序は、**コード内でのインポート順**に直接依存する。

### 本番ビルド時の最適化

| 項目 | 詳細 |
|---|---|
| チャンク化 | CSS ファイルは自動的に連結・最適化される |
| コード分割 | 各ルートに必要な最小限の CSS のみが読み込まれる |

### 推奨される管理プラクティス

- **インポートの集約：** CSS インポートはルートレイアウトなど単一のエントリファイルに限定する
- **一貫した命名：** `<name>.module.css` のような統一した命名規則を採用する
- **自動ソートの無効化：** `sort-imports` など、インポート順を自動変更するリンターはオフにする
- **詳細設定：** `next.config.js` の `cssChunking` オプションでチャンク化の挙動を制御できる

## 開発環境と本番環境の動作差異

| | 開発環境（`next dev`） | 本番環境（`next build`） |
|---|---|---|
| スタイルの反映 | Fast Refresh により即座に反映 | ビルド時に確定 |
| JS の要否 | Fast Refresh に JavaScript が必須 | JavaScript 無効でも CSS は読み込まれる |
| CSS の処理 | そのまま提供 | 縮小（minify）＋コード分割 |

> CSS の順序が開発環境と本番環境で異なる可能性があるため、最終確認は必ずビルド（`next build`）で行うこと。
