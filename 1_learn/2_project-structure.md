#### 並行ルートと交差ルート

> これらの機能は、スロットベースのレイアウトやモーダルルーティングなど、特定のUIパターンに適合します。
> 親レイアウトによってレンダリングされる名前付きスロットに使用します@slot。
> インターセプトパターンを使用すると、URLを変更せずに現在のレイアウト内に別のルートをレンダリングできます。
> たとえば、詳細ビューをリストの上にモーダルとして表示する場合などに使用できます。

| パターン（ドキュメント） | 意味 | 典型的な使用例 |
|---|---|---|
| `@folder` | 名前付きスロット | サイドバー＋メインコンテンツ |
| `(.)folder` | 同じレベルでインターセプト | 兄弟ルートをモーダルでプレビューする |
| `(..)folder` | 親をインターセプトする | 親の子をオーバーレイとして開く |
| `(..)(..)folder` | 2つのレベルをインターセプト | 深く入れ子になったオーバーレイ |
| `(...)folder` | ルートからのインターセプト | 現在のビューで任意のルートを表示する |

```bash
├── (shop)
│   ├── layout.js

│   │   └── page.js
│   └── cart → /cart
│       └── page.js
└── checkout → /checkout
    └── page.js
```

> これは、アプリケーションをUIやユーザーエクスペリエンスが完全に異なるセクションに分割する場合に便利です。
> 各ルートレイアウトには、<root>タグ<html>と<root><body>タグを追加する必要があります。

```bash
app
├── (marketing)
│   ├── layout.js
│   └── ...
└── (shop)
    ├── layout.js
    └── ...
```

> 上記の例では、と の両方(marketing)が(shop)独自のルートレイアウトを持っています。
