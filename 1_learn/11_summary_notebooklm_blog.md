# Next.js 16 でCSS迷子にならないための5つの真実

## 1. グローバルCSSは「一度読み込むと消えない」

Next.js は React の Suspense 統合により、**ルート間をナビゲートしてもスタイルシートが削除されない**。特定ページ向けのグローバルスタイルが意図せず他ページのデザインを壊すリスクがある。

**対策：** グローバル CSS の使用は最小限に留め、コンポーネントのスタイリングには Tailwind CSS または CSS Modules を使う。

| 用途 | 推奨手法 |
|---|---|
| 本当にグローバルなスタイル（リセット CSS 等） | Global CSS |
| コンポーネントのスタイリング | Tailwind CSS |
| カスタムスコープが必要な場合 | CSS Modules |

---

## 2. CSS の優先順位を決めるのは「インポートの順番」

CSS の適用順序は、**コード内でのインポート順に直接依存する**。共通コンポーネントをページ固有 CSS より先にインポートすれば、共通コンポーネントのスタイルが先に読み込まれる。

**注意点：** ESLint の `sort-imports` などインポートを自動ソートするリンター・フォーマッターは**オフにする**こと。アルファベット順への自動整列が本番環境のデザイン崩れの原因になりうる。

---

## 3. React 19 がもたらす「外部スタイルシート」の新しい形

Next.js 16 × React 19 の統合により、コンポーネント内で標準的な `<link rel="stylesheet">` タグが使えるようになった。

**メリット：**
- サードパーティ製スタイルの扱いが簡潔になる
- コンポーネントと同じ場所に CSS の参照を配置できる（コロケーション）
- グローバル汚染のリスクを抑えつつ、スタイルをコンポーネントに閉じ込められる

---

## 4. 開発環境と本番環境で CSS の挙動が異なる

| | 開発環境（`next dev`）| 本番環境（`next build`）|
|---|---|---|
| CSS の処理 | そのまま提供 | 連結・縮小・チャンク化 |
| JS の要否 | Fast Refresh に必須 | JS 無効でも CSS は読み込まれる |
| 適用順序 | 開発時の順序 | 最適化により微妙にズレる可能性あり |

**鉄則：** 本番ビルド（`next build`）の結果こそが唯一の真実。必ずビルド後の状態で最終確認する。チャンク化の挙動を調整したい場合は `next.config.js` の `cssChunking` オプションを使う。

---

## 5. 公式が示すスタイリングの「黄金律」

```
メイン    → Tailwind CSS      一般的なデザインパターンの大半をカバー
補足      → CSS Modules       Tailwind で不十分な場合のみ
最小限    → Global CSS        layout.tsx でのみインポート、基盤スタイルに限定
```

**運用のコツ：**
- CSS Modules のファイル名は `<name>.module.css` に統一する
- 共有スタイルは共有コンポーネントに抽出し、インポートの重複を避ける
- CSS インポートは可能な限り単一のエントリファイルに集約する

---

## まとめ

Next.js 16 における CSS 管理は「見た目」の問題ではなく、**パフォーマンスとルート遷移の予測可能性を管理するエンジニアリング**。インポート順ひとつにも意図を持ち、本番ビルドで必ず挙動を確認する習慣が、プロジェクトの安定性を高める。
