> レイアウトとは、複数のページ間で共有されるUIのことです。

へーそうなのか。  
コンポーネント自体再レンダーされないってこと…？  
→ イメージした再レンダーされないではなかった。  

> 「再レンダリングされない」の意味
> これは「ページ遷移時にレイアウト自体が再マウント・再レンダリングされない」という意味です。

### ネストされたルートの作成

> ネストされたルートとは、複数のURLセグメントで構成されるルートのことです。例えば、この/blog/[slug]ルートは3つのセグメントで構成されています。

- /（ルートセグメント）
- blog（セグメント）
- [slug]（葉片）

ふむ。  

> Next.jsでは、以下の仕組みでルーティングを構成します。

> ・フォルダを使って、URLのセグメントに対応するルートセグメントを定義します。
> ・ファイル（page.tsxやlayout.tsxなど）を使って、そのセグメントで表示されるUIを作成します。
> ネストされたルートを作成するには、フォルダの中にフォルダを入れ子にします。例えば、/blog というルートを追加したい場合は、appディレクトリの中に blog というフォルダを作成します。次に、/blog に外部からアクセスできるようにするために、その中に page.tsx ファイルを追加してください。

### ネストされたレイアウト

> デフォルトでは、フォルダ階層内のレイアウトもネスト（入れ子）構造になります。
> つまり、childrenプロップを通じて子レイアウトをラップする仕組みです。
> 特定のルートセグメント（フォルダ）の中にレイアウトファイルを追加することで、レイアウトをネストさせることができます。

> 例えば、/blogルート用のレイアウトを作成するには、blogフォルダの中に新しいレイアウトファイルを追加してください。

`app/blog/layout.tsx`として作成すると。  

```ts
export default function BlogLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return <section>{children}</section>
}
```

こうすると、`<section></section>`の中に`page.tsx`の中身が出てくると。  
で、親のレイアウトもネストしているのでちゃんと`<body>`とかのタグも出ていると。  

> 上記の2つのレイアウトを組み合わせると、ルートレイアウト（app/layout.js）がブログレイアウト（app/blog/layout.js）を囲み、さらにそのブログレイアウトがブログ一覧ページ（app/blog/page.js）とブログ記事ページ（app/blog/[slug]/page.js）を囲む形になります。


### 動的なセグメントを作成する

> 動的セグメントを使用すると、データに基づいてルートを生成できます。
> たとえば、個々のブログ記事ごとに手動でルートを作成する代わりに、ブログ記事のデータに基づいてルートを生成する動的セグメントを作成できます。

なるほど。  

> 動的セグメントを作成するには、セグメント（フォルダー）名を角括弧で囲みます：[segmentName]。例えば、app/blog/[slug]/page.tsx というルートの場合、[slug] が動的セグメントになります。  

あーこれはチュートリアルであったね。  

> ダイナミックセグメントとparamsプロップスについて詳しく学びましょう。
> ダイナミックセグメント内のネストされたレイアウトでも、paramsプロップスにアクセスすることが可能です。

なるほど。  
`[slug]` としてアクセスした`params`については`props`で取得できるのか。  

`app/blog/[slug]/page.tsx`に実装してみる。  

```ts
export default async function Page({ params }: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return (
    <h1> Blog!!! params: {slug}</h1>
  );
}
```
としてつくってみた。  

### 検索パラメータを使用したレンダリング

> サーバーコンポーネントページでは、以下のプロパティを使用して検索パラメータにアクセスできますsearchParams。

なるほど、 `searchParams`で検索パラメータにアクセスできると。  

`app/page.tsx`を修正してみる。  


```ts
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
  return (
    <>
      <h1>Top Page!</h1>
      <p>searchParams: {filters}</p>
    </>
  );
}
```
こんな感じで作成。  

<http://localhost:3000/?filters=hoge>みたいな感じでアクセスしてみる。  

```
Top Page!
searchParams: hoge
```
って感じで表示される。  


```
const { keyword, sort, page } = await searchParams
```

みたいな感じにすれば、`keyword`, `sort`, `page`って感じでパラメータを受け取れるとのこと。  


> searchParamsを使用すると、そのページは動的レンダリング（dynamic rendering）へと切り替わります。
> これは、検索パラメータを読み取るためにリクエストの受信が必要になるためです。
> クライアントコンポーネントでは、useSearchParamsフックを使用して検索パラメータを読み取ることができます。
> プリレンダリングされたルートおよび動的にレンダリングされるルートにおけるuseSearchParamsの詳細については、こちらをご覧ください。

なるほど。

#### 何を使うべきか その使い分け

> ページのデータを読み込むために検索パラメータが必要な場合（例：ページネーション、データベースのフィルタリングなど）は、searchParamsプロップを使用してください。 
> 検索パラメータをクライアント側だけで使用する場合（例：プロップ経由で読み込み済みのリストをフィルタリングするなど）は、useSearchParamsを使用してください。
> ちょっとした最適化として、コールバック関数やイベントハンドラ内で new URLSearchParams(window.location.search) を使用すれば、再レンダリングを発生させることなく検索パラメータを読み取ることができます。

なーる。  

### ページ間のリンク

> ルート間を移動するには、<Link> コンポーネントを使用します。 
> <Link> は Next.js 組み込みのコンポーネントで、HTML の <a> タグを拡張し、プリフェッチ機能やクライアントサイド・ナビゲーションを提供します。
> 例えば、ブログ記事の一覧を生成するには、next/link から <Link> をインポートし、href プロップをコンポーネントに渡します。

なるほど、`<Link>`コンポーネントっていうのが用意されていると。  

```ts
import Link from 'next/link'
 
export default async function Post({ post }) {
  const posts = await getPosts()
 
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.slug}>
          <Link href={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  )
}
```

が例。  

自分でも作ってみる。  

`app/page.tsx`を以下にする。  
```ts
import Link from 'next/link'

export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
  const slug = "abcdefg"
  return (
    <>
      <h1>Top Page!</h1>
      <p>searchParams: {filters}</p>
      <Link href={`/blog/${slug}`}>こちらへどうぞ!</Link>
    </>
  );
}
```

できたと。

> より複雑な遷移が必要な場合は、useRouterフックを使用することもできます。

### ルートプロップヘルパー

> Next.jsでは、ルート構造からparams（パラメーター）や名前付きスロットを推論するためのユーティリティ型が用意されています。
> - PageProps：ページコンポーネント用のプロップス。paramsやsearchParamsが含まれます。
> - LayoutProps：レイアウトコンポーネント用のプロップス。childrenや、名前付きスロット（@analyticsのようなフォルダなど）が含まれます。
> これらはグローバルに利用可能なヘルパーで、next dev、next build、またはnext typegenの実行時に生成されます。

なるほどなー。  
`PageProps`の例は以下。
